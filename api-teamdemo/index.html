Request Builder — Try Breaking the API

Configure three things about an inbound notification call from UKG to EN, then send it. Each setting maps to one specific kind of failure — flip them on purpose and watch how the status code changes.

POST qa.employeenavigator.net
/API/Payroll/companies/AMS123/employees/EMP-10042/notification
— — Configure your request, then click Send.
Each setting maps to one check the API performs. Flip one to "fail" and you'll see exactly which check failed — and which status code that produces.
// Response body will appear here
The shortcut for triage: 401 = something wrong with the token (auth). 403 = partner hasn't enabled their side of the integration (no access to their system). 400 = the request body is malformed or rejected for data reasons. Recognizing this trio at a glance is most of the job.
